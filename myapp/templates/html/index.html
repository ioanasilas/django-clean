Project Overview

As part of our academic journey and with the support of an IBM-led mentorship program, we developed a fully functional CI/CD (Continuous Integration / Continuous Delivery) pipeline. The goal was to simulate a real-world DevOps workflow using modern tools and technologies that professional development teams rely on today.

The pipeline was deployed in a virtual environment running Rocky Linux 9 and relied on Jenkins for automation, Podman for containerization, and GitHub as the version control platform. We also used Docker Hub as a public registry to store our container images.

What We Built

  • Configured a Jenkins pipeline to automatically clone the project repo from GitHub.
  • Built the container image using Podman, tagged it appropriately, and ran it for testing.
  • Automatically stopped and deleted the test container after validation.
  • Successfully pushed the final image to Docker Hub using Podman.
  • Added a custom API endpoint returning JSON with project and team info to show meaningful changes to the app.
  • Tested the container independently by pulling it from Docker Hub and running it locally with Podman.

Beyond technical skills, we improved our collaboration and communication through version control, regular sync-ups, and problem-solving together as a team.

Technologies Used

  • Jenkins – for automating the CI/CD pipeline
  • Podman – for building, running, and managing containers without needing root access
  • GitHub – as the source code repository and version control platform
  • Docker Hub – for hosting and distributing container images
  • Rocky Linux 9 – as the base OS for our development environment

What We Learned

This project gave us hands-on experience with real-world DevOps tools and workflows. We learned how to:

  • Automate code integration, testing, and deployment using Jenkins
  • Work with containerized applications using Podman
  • Manage repositories and understand CI/CD triggers through GitHub
  • Push and pull container images between local environments and remote registries
  • Troubleshoot build and deployment issues in a Linux-based environment

Beyond technical skills, we improved our collaboration and communication through version control, regular sync-ups, and problem-solving together as a team.
